Lab1 – Python, Numpy, Plotting

Much of machine learning today is done in the Python programming language. It is very important to get good at Python and at using key packages.

Lab1 involves a lot of review. Most students will probably not finish in the allotted lab time, so start ahead of time if you can so that you have good questions for the TAs. The TAs will help you get up to speed.

Python setup

You will use Anaconda Python 3.8 for this course, which is standard. (You can also use Python 3.9, but I recommend against it because some libraries still have compatibility issues.)

If Python is installed, you can check the version:

$ python --version
Python 3.8.X

If you are using Concordia's lab computers, Anaconda Python is already installed with all the necessary dependencies for this course. You may need to load the correct version of Python using this AITS script:

$ module load anaconda3/2021.05

Personal computers

If you do not have Anaconda Python 3.8 installed and cannot upgrade to it, follow these steps.

Download and install Miniconda Python 3.8 for your operating system. The installation details depend on your operating system. For example, if the installer asks whether you want it to run "conda init" say yes. If you are installing from a bash shell, you may need to close it and start a new one for the new conda command to appear in your PATH. Use the where (on MacOS/Linux) or which (on Windows) command to check:

$ which conda
/home/username/miniconda3/bin/conda

However you install, you need the new "conda" command to be in your PATH so that you can run it.

Ensure the Numpy, Matplotlib, and Jupyter packages are installed:

$ conda install numpy matplotlib jupyter

Installing the packages may take a few minutes.

Start a Jupyter Notebook instance

The lab exercises are given as a Jupyter Notebook. To open the notebook, run the following in a terminal:

$ cd <directory where you unzipped lab1>
$ jupyter notebook

This should automatically open a web browser and you should see the following:

Click on lab1-exercises.ipynb to begin. Once you've opened the notebook, continue reading below.

1. Python language proficiency

Look through this Python language review written specifically for this course. It is too long to read during lab time, but you should spend ~10 minutes looking through it right now. If anything is confusing, make a mental note: "To do well in this class, I need to get better at Python." If you have time, ask TAs about any examples you do not understand.

Lab exercises

Switch back to the lab1-exercises.ipynb that you opened earlier and complete exercises 1.1–1.4. If you struggle to complete these on your own, you will need to invest extra time to improve your Python.

2. Numerical computing with Numpy

Look through the Numpy review written specifically for this course. The review materials are much too long to read during lab time, but you should spend ~10 minutes looking through it right now. If you find it unfamiliar or challenging, then you must invest extra time to learn how to use Numpy properly. What you learn will directly translate to how other array-based frameworks work, like PyTorch, TensorFlow, or JAX.

Lab exercises

Switch back to lab1-exercises.ipynb and complete exercises 2.1–2.3. If you struggle to complete these on your own, move on and make time outside of lab to improve your knowledge. You should at least be comfortable with creating Numpy arrays before moving on to plotting.

3. Plotting with Matplotlib

Once you understand Numpy you can learn how to make plots with the Matplotlib package. There are other good plotting packages in Python, but you should start with this one. Matplotlib provides a submodule called pyplot that works similarly to Matlab. The official Pyplot tutorial is the easiest way to get started. You may also find this short Pyplot tutorial from Stanford's cs231n course to be helpful.

Lab exercises

Switch back to lab1-exercises.ipynb and complete exercises 3.1–3.3.

You're done!!

Optional: Tabular data with Pandas

Finally, if you are familiar with data frames in the R programming language, then you may enjoy learning how to use the pandas package in Python. Pandas provides a DataFrame type that helps with productivity and is displayed beautifully within Jupyter Notebooks. Use of pandas is optional in this course and is not covered.
