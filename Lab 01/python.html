Python Language Review

This document is a series of short Python code examples. It was written for COMP 6321. You need to understand these language features to be a half-competent Python programmer. If you do not understand these things, you will struggle to write machine learning code. You will also probably say something stupid during a coding interview.

  • If you're relatively new to the language, you should use a tutorial like this to broaden your knowledge.
  • For the sake of your career, you are strongly advised to learn the coding style described in PEP8 and documentation style in PEP257. However, you will not lose marks in this class for failing to do so.
  • The code descriptions below are deliberately suggestive of how Python really works. For example, rather than describing the statement x = 3 as "set x to 3" we describe it as "create a new variable 'x' and make it refer to an integer object that holds value 3."

Lines starting with >>> represents the standard Python interactive REPL prompt:

$ python      # From a terminal, start the Python interpreter in interactive mode
>>> x = 42    # Create a new variable 'x' and make it refer to an integer object that holds value 42
>>> x         # Print whatever 'x' is, in interactive mode
42
>>> print(x)  # Print whatever 'x' is, even if we weren't in interactive mode
42
>>> quit()    # Quit Python interpreter back to terminal
$

Jupyter Notebooks don't have the >>> prompt, but each code cell acts as if you were typing the lines of code into a single prompt.

Variables

A Python variable is a symbol that refers to an object. Never confuse the symbol (the variable name) with the object that it refers to. They are two different things. If you fail to understand this, you will live in a world of confusion.

>>> x = 5    # Create a new variable 'x' and make it refer to an integer object that holds value 5
>>> y = x    # Create a new variable 'y' and make it refer to the same integer object as 'x'
>>> x += 10  # Make 'x' refer to a new integer object with value (x + 10)
>>> x        # Variable x now refers to the new integer object, with value 15
15
>>> y        # Variable y still refers to the old integer object, with value 5
5

One way to see how variables work is to understand the difference between the == and is operators:

>>> x = 257     # Make 'x' refer to an integer object that holds value 257
>>> x == 257    # Does x refer to an object that has SAME VALUE as this other integer object?
True            # Yes!
>>> x is 257    # Does x refer to EXACT SAME OBJECT as this other integer object?
False           # No! There are two different integer objects in memory, each with value 257.

The result would be different for integers 0..256 because Python tries to save memory by pooling small integers. Likewise for short strings. If you find this confusing, ask a TA for insight.

If you know the C programming language, think of Python variables as pointers (int* rather than int). When you make a new variable, it can point to an existing object, or it can point to something new, but never confuse the pointer (the variable) with the object that it refers to.

Numbers

Know the difference between float division (the default) and integer division:

>>> 3 / 2           # In Python 3, dividing two integers results in a floating-point value
1.5
>>> 3 // 2          # In Python 3, you must use the // operator to force integer division
1

You should know about the ** exponentiation operator:

>>> 2**3            # 2 cubed
8
>>> 2**0.5          # square root of 2
1.4142135623730951

You should know how to express floating-point numbers (float) in scientific notation:

>>> 1e-3            # Same as 10**(-3), but prefer scientific notation
0.001

>>> 1e3             # A number expressed in scientific notation is always of type float
1000.0

Python floats are 64-bit, like a double in C. You should know about the limitations of floats:

>>> 1e-323 == 0.0   # Very very tiny value
False
>>> 1e-324 == 0.0   # So tiny we can't represent it with 64-bit float anymore, rounds down to 0.0
True
>>> 1 / 2**0.5      # Mathematically, 1/sqrt(2) == sqrt(2)/2, but ...
0.7071067811865475
>>> 2**0.5 / 2      # ... *numerically* they are slightly different
0.7071067811865476

If you don't understand the above, read more about IEEE floating-point numbers.

Strings

You should know the different ways to create strings:

>>> s = 'hello'      # Make a new variable 's' and make it refer to a string object with value 'hello'
>>> s = "hello"      # Same thing, with double-quotes
>>> s = """hello"""  # Same thing, with triple-quotes
>>> s
'hello'              # By default Python displays strings with single quotes around them
>>> print(s)
hello                # Printing a string does not result in surrounding quotes

The triple-quote is special because it lets you define multi-line strings:

>>> print('first\nsecond')  # Single quotes require escape character \n (newline)
first
second

>>> print("""first
... second""")              # Triple-quotes let you write multiple lines of text right into the string
first
second

You can treat strings like a sequence of characters:

>>> s = 'abcd'
>>> s[2]         # Get the character at index 2 (the third character)
'c'
>>> len(s)       # Get the length of the string (the number of characters)
4

You can append strings together:

>>> s = 'before ' + s + ' after'  # Make 's' refer to a new string object built from the old value
>>> s
'before abcd after'

You can get a string representation of most Python objects:

>>> x = 123
>>> str(x)
'123'

You can also split and join strings together:

>>> words = 'Dog versus cat'.split()     # Split a string by whitespace
>>> words
['Dog', 'versus', 'cat']                 # The result is a list of string objects

>>> '-'.join(words)                      # Make a string object with value '-' and then call its join()
'Dog-versus-cat'                         # method to build a new string from a list of other strings

Knowing what string functions Python provides will help you write better code.

String formatting

See this Python string formatting guide for lots of examples, comparing both "old style" and "new style."

The "old style" string formatting in Python is similar to the printf function in C:

>>> name = 'Jack'
>>> age = 7
>>> '%s is %d years old' % (name, age)
'Jack is 7 years old'

The formatting scheme is similar to the printf function in C, where %s means string, %.2f means "floating point value with 2 decimals of precision," etc.

Lists

A list is a mutable sequence.

You should know how to make lists:

>>> x = []                   # Make 'x' refer to a new empty list object
>>> x = list()               # Same as above
>>> x = [3, 'abc']           # Make 'x' refer to a new list object that has two slots, one referring
                             # to an integer object and one referring to a string object

You can get and set items by index:

>>> x[0]         # Get the reference in the 1st slot
3

>>> x[0] = 7     # Set the reference in the 1st slot to a new integer object
>>> x
[7, 'abc']

You should think of lists as containing 'slots', where each slot is a reference to an object. Make sure you understand the following example:

>>> x = 257            # Make a variable 'x' that refers to an integer object
>>> y = [x, x, 257]    # Make a list object where each of the slots refers to an integer object,
                       # and make a variable 'y' refer to that list
>>> y
[257, 257, 257]        # By all appearances, a list of three identical integers

>>> y[0] is y[1]       # The first two slots refer to the same integer object
True

>>> y[0] is y[2]       # The last slot refers to a different integer object, though it has the same value
False

If you know C programming, think of each slot as a pointer to an object. If you run the above code, the Python interpreter allocates the following chunks of memory:

Arrows represent a reference. The list object comprises a fixed-size chunk and a variable-sized chunk of slots (the items). In this example there are two distinct integer objects, even though they both hold value 257.

You can add or remove items from a list:

>>> x = ['a', 'b', 'c']    # Create a list object with 3 slots
>>> x.append('d')          # Create a 4th slot and make it refer to a new string
>>> x
['a', 'b', 'c', 'd']

>>> del x[1]               # Delete the 2nd slot, removing the reference and shifting the later slots
>>> x
['a', 'c', 'd']

>>> x.pop()                # Remove the last slot in the list, and return whatever object it refered to
'd'
>>> x
['a', 'c']

You can concatenate lists with + or +=:

>>> x = ['a', 'b']
>>> y = ['c']
>>> x + y           # Create a new list with 3 slots that refer to the same objects as in 'x' and 'y'
['a', 'b', 'c']

You can repeat a list with * or *=:

>>> x * 3
['a', 'b', 'a', 'b', 'a', 'b']

You can build a list from any sequence-like object, such as a string:

>>> list('hello')          # Create a list of five string objects, one for each character
['h', 'e', 'l', 'l', 'o']

Since a list slot can refer to any kind of object, you can also make list-of-lists:

>>> x = [[1, 2, 3], ['foo', 'bar']]   # Make three lists: two inner lists, and one outer list
>>> x[0]                              # Get list object referred to by the first slot
[1, 2, 3]
>>> x[1][0]                           # Get string object referred to by 2nd list's 1st slot
'foo'

Immutable types

An object is immutable if its value cannot be changed after initialization. It is read-only after creation.

That sounds like a bad thing. Why restrict objects from changing their value? Two reasons:

  1. Immutable objects requires less memory than a mutable type.
  2. Immutable objects can be safely used as dictonary keys: they can't be invalidated by a change in value.

For example, strings are immutable. You can't modify a string, you can only use it to build new strings:

>>> s = 'cat'
>>> s[0] = 'h'
TypeError: 'str' object does not support item assignment

>>> s.replace('c', 'h')  # Returns a new string object built from the old one
'hat'
>>> s                    # The old string object keeps its initial value
'cat'

We often use strings as keys in a dictionary or a set (covered later), so immutability is important for strings.

Tuples

A tuple works the same as a list, except that tuples are immutable. You can't add/remove/replace items.

Tuples are being created and destroyed all the time in Python. Each time you call a function, a tuple is created to hold its arguments. Each time you return multiple values from a function, a tuple is created to hold them.

You should know how to make tuples:

>>> x = ()           # Make 'x' refer to an empty tuple
>>> x = tuple()      # Same as above
>>> x = (1, 2, 3)    # Make 'x' refer to a new tuple that has 3 slots, each referring to an integer object
>>> x = 1, 2, 3      # Same as above
>>> x
(1, 2, 3)

When you assign multiple variables at once, you're using tuples:

>>> x, y = 3, 5   # Create a tuple (3, 5) and unpack the values into tuple (x, y)
>>> x
3
>>> y
5

The idiomatic way to swap two variables in Python uses tuples:

>>> x, y = y, x    # Create a tuple (y, x) and unpack the values into tuple (x, y)
>>> x, y
(5, 3)

Tuples have a length:

>>> x = (0, 1, 2)
>>> len(x)
3

Tuples are slightly more memory efficient than lists (one chunk, not two), but this seldom matters.

Indexing and slicing

Strings, lists, tuples, and most sequence-like objects support indexing and slicing exactly the same way.

You should know how slicing works:

>>> x = 'abcd'
>>> x[1:3]        # New string object copied from index 1 and stopping at index 3
'bc'

>>> x[2:]         # Same as x[2:len(x)]
'cd'

>>> x[:2]         # Same as x[0:2]
'ab'

>>> i, n = 1, 3
>>> x[i:i+n]      # Same as x[1:4]
'bcd'

Beware that Python will not raise an error if you ask for a slice outside the range of items:

>>> x[1:100]      # OK, but don't expect 99 items
'bcd'

>>> x[50:100]     # OK, but don't expect 49 items
''

You should also know how negative indexing works. A negative index -i always refers to slot len - i:

>>> x[-1]     # Same as x[len(x)-1]
'd'

>>> x[-2:]    # Same as x[len(x)-2:]
'cd'

>>> x[1:-1]   # Same as x[1:len(x)-1]
'bc'

The result of a slice is always a new object (string, list, tuple), as you can see below:

>>> x = [1, 2, 3]   # Create a new list with 3 slots
>>> y = x[:]        # Create a new list where all the slots refer to the same objects as 'x'
>>> x is y          # Now 'x' and 'y' are different list objects
False
>>> x[0] is y[0]    # But their slots refer to the same integer objects
True

Dictionaries

A dictionary is a set of (key, value) pairs. Given a key, it returns the corresponding value object very fast.

>>> grades = {'Jack': 85, 'Jill': 92}  # Create a new dict object with two (key, value) pairs
>>> grades['Jack']                     # Find a matching key and return the associated value
85

Dictionaries are being created and access all the time in Python. Every time you use a variable by name (the key), the Python interpreter looks up which object it refers to (the value) in a special dictionary object:

>>> x = [1, 2, 3]           # Make a new variable 'x' that refers to a list object
>>> locals()                # Get the internal dictionary that Python uses to look up variables
{..., 'x': [1, 2, 3]}       # Aha! Here's where Python keeps track of variable 'x'!

>>> del x                   # We can even delete the 'x' entry from the dictionary of variables
>>> locals()                # The 'x' entry is now gone from the dictionary of variables
{...}                       
>>> x                       # Now the 'x' variable is gone!
NameError: name 'x' is not defined

You should know how to create dictionary objects of your own:

>>> x = {}                            # Create an empty dict object 
>>> x = dict()                        # Same as above

>>> x = {'a': 1, 'b': 2}              # Create a dict object with two (key, value) pairs
>>> x = dict([('a', 1), ('b', 2)])    # Same as above, using a list of tuples

Dictionaries are mutable. You can add or remove entries from a dictionary:

>>> x['c'] = 3        # Set the value associated with key 'c', creating a new entry if necessary
>>> del x['a']        # Delete the entry with key 'a' from the dictionary
>>> x
{'b': 2, 'c': 3}

You can test whether a key is in a dictionary, very efficiently:

>>> 'b' in x   # There is one value
True
>>> 'a' in x   # No key has value 'a' anymore (it was deleted)
False

You can merge two dictionaries:

>>> x = {'a': 1, 'b': 2}    # Create a dict object with two entries
>>> y = {'b': 8, 'c': 9}    # Create another dict object
>>> x.update(y)             # Merge the entries from 'y' into the entries from 'x'
>>> x
{'a': 1, 'b': 8, 'c': 9}    # The value associated with key 'b' was replaced

Dictionaries are fast because they are are implemented as hash tables. That means a key must be hashable:

>>> hash('cat')
2977299099912581022

Tuples of hashable objects are hashable. Lists, dicts and sets are not hashable and cannot be used as keys:

>>> hash(tuple())
3527539

>>> hash(list())
TypeError: unhashable type: 'list'

>>> hash(dict())
TypeError: unhashable type: 'dict'

>>> hash(set())
TypeError: unhashable type: 'set'

Sets

A set is a dictionary of just keys, with no associated values. They are useful for finding unique items in a list, and for keeping track of whether you've seen a particular key before.

You shoud know how to make sets:

>>> x = set()           # Create an empty set object
>>> x = {1, 2}          # Create a set object with two keys, both of which are integer objects
>>> x = set([1, 2])     # Same as above

Testing whether a key is in a set is very fast, much faster than testing whether an item is in a list:

>>> 1 in x
True
>>> 3 in x
False

They work like mathematical sets, with union and intersection operations:

>>> y = {2, 3}
>>> x.union(y)            # {1, 2} union {2, 3}
{1, 2, 3}
>>> x - y                 # {1, 2} minus {2, 3}
{1}
>>> y - x                 # {2, 3} minus {1, 2}
{3}

If you build one from a sequence, it keeps only the unique values:

>>> set([3, 1, 2, 2, 1])
{1, 2, 3}

You can turn a set into a list or a tuple. Just be aware that the order of items is arbitrary:

>>> list({'a', 'b', 'c'})
['b', 'a', 'c']

Conditionals

You can write if-statements using standard conditional expressions (==, !=, <, >, etc) and also expressions more unique to Python (in, is):

>>> x = {1, 2}
>>> if 3 in x:
...     print("yep!")
... else:
        print("nope")
nope

You can combine or modify conditional expressions with logical operators (and, or, not):

>>> (3 not in x) and (len(x) == 2)
True

You can chain if-else statements together with elif:

>>> if 3 in x:
...     print("yep!")
... elif len(x) == 10:
...     print("this?")
... else:
...     print("nope")
nope

You can also use a special if-else expression, which has the structure yes_value if condition else no_value:

>>> y = 'yep!' if 2 in x else 'nope'   # Since integer 2 is in x, return the expression BEFORE the if
>>> y
'yep!'

>>> y = 'yep!' if 3 in x else 'nope'   #
>>> y
'nope'

This is similar to C conditional expressions, where one can write condition ? yes_value : no_value.

You should have enough programming experience that if-statements are trivial. If you are puzzled by the above code, you should probably drop the course because you're starting too far behind.

Iterators

Iterators provide a sequence of values, without having to build a list of values in memory. They are used all the time in Python, especially Python 3, and not understanding them will lead to confusion.

A range is the most basic iterator:

>>> x = range(1000)  # Create a range object (an type of iterator) representing values (0, 1, ..., 999)
>>> x
range(0, 1000)       # Since 'x' refers to a range object, printing it tells us more about the object

Think of this as a "virtual tuple" holding 1000 integers, except it doesn't waste memory on creating 1000 actual integer objects.

We can treat a range object as if it were an actual tuple:

>>> len(x)
1000
>>> x[500]
500

And, if we really want to, we can use the iterator to build an tuple object containing 1000 integer objects:

>>> tuple(x)
(0, 1, 2, ... 999)

The whole point of range objects, and of iterators in general, is to make looping faster and memory efficient.

The zip function is another important one to understand:

>>> x = ['a', 'b', 'c']
>>> y = [1, 2, 3]
>>> z = zip(x, y)   # Create an iterator object that will return a sequence of tuples (x[i], y[i])
>>> list(z)         # Create a list object and fill each slot with the next object returned by 'z'
[('a', 1), ('b', 2), ('c', 3)]

This will be very helpful for writing concise, efficient loops.

The reversed function also return an iterator, without building a new sequence in memory.

Loops

Python loops are pretty standard. There are for-loops and while-loops.

For-loops repeat once for each item in the given sequence:

>>> for i in (0, 1, 2):  # Make variable 'i' refer to the next object in the sequence each time 
...     print(i)
0
1
2

For-loops work with iterators too:

>>> for i in range(3):   # Make variable 'i' refer to the next object returned by the iterator until done
...     print(i)
0
1
2

While-loops work in the usual way:

>>> i = 0          # Make variable 'i' refer to the integer object with value 0
>>> while i < 3:   # While the condition is true, run the body of the loop
...     i += 1     # Make variable 'i' refer to a new integer object that has value i + 1
>>> i              # Variable 'i' refers to whatever object it did when the loop terminated
3

Beware that loops do not have their own lexical scope. Variables created inside a loop persist afterwards:

>>> for i in range(3):
...    j = i            # Make new variable j refer to same object as i currently does
>>> j                   # Variable j refers to whatever object it did when the loop terminated
2

You should know how the break keyword works, just like in C programming:

>>> for i in range(100):
...    if i >= 3:
...        break
>>> i
3

You should know how the continue keyword works, just like in C programming:

>>> for i in range(100):
...    if i >= 3:
...        continue
...    print(i)
0
1
2

It may be helpful to know that for-loops support an else clause, which only runs if the loop successfully completed all iterations without breaking out early:

>>> for i in range(3):
...     if i > 100:          # Always be False, so all three loop iterations will 'complete'
...         break
... else:
...     print("All iterations completed")
All iterations completed

You should know how to iterate over parallel lists using the zip function:

>>> names = ['Jack', 'Jill']
>>> grades = [85, 92]
>>> for name, grade in zip(names, grades):   # Loop (name, grade) over each pair (names[i], grades[i])
...    print(name, grade)
Jack 85
Jill 92

This is often tidier than writing a loop that tries to access names[i], grades[i] by integer index.

You can likewise iterate over the (key, value) pairs in dictionary this way:

>>> grades = {'Jack': 85, 'Jill': 92}
>>> for name, grade in grades.items():
...     print(name, grade)
Jack 85
Jill 92

List comprehensions

List comprehensions are a convenient and concise way of building lists from other sequences:

>>> x = [i*2 for i in range(4)]   # Create a new list and initialize each slot using a loop
>>> x
[0, 2, 4, 6]

You can put whatever expression you want in a list comprehension:

>>> ["%.2f" % (i/3) for i in range(4)]    # Fill each slot with the result of a string formatting operation
['0.00', '0.33', '0.67', '1.00']

You can also filter values from the new list:

>>> [i for i in range(8) if i % 2 == 0]   # Create a list object, but only add a new slot if 'i' is even
[0, 2, 4, 6]

You can also create dictionaries (or sets) with list comprehension syntax:

>>> names = ['Jack', 'Jill']
>>> grades = [85, 92]
>>> x = {name: grade for name, grade in zip(names, grades)}   # Create (key, value) from each (name, grade)
>>> x
{'Jack': 85, 'Jill': 92}

You can have nested loops:

>>> [(i, j) for i in range(2)
...         for j in range(3)]
[(0, 0), (0, 1), (0, 2),
 (1, 0), (1, 1), (1, 2)]

This can be confusing. Remember that the earlier for-loops are the 'outer' ones, as if it were written as:

for i in range(2):
    for j in range(3):
        (i, j)

Functions

You should know how to use the following built-in functions:

  • len returns the length of any sequence-like object
  • abs returns the absolute value of its argument
  • any returns True if any item in the sequence evaluates as True
  • all returns True if all items in the sequence evaluate as True
  • enumerate iterates over a sequence, returning a tuple (index, item) for each item
  • isinstance returns True if the object is an instance of the given type
  • type returns the type of the argument object
  • max returns the object with maximum value from sequence
  • min returns the object with minimum value from sequence
  • sum returns the sum of all objects in a sequence
  • open opens a file for reading and/or writing
  • range iterates over a range of integers, returning an integer object for each one
  • print prints a string to the terminal
  • filter iterates over a sequence, returning only those items for which a function evaluates to True
  • map iterates over a sequence, returning the result of applying a function to each item
  • sorted returns a list object containing the sequence items in sorted order
  • reversed iterates over a sequence in reverse order
  • zip iterates over multiple sequences (x, y, ...) in parallel, returning a tuple (x[i], y[i], ...) for each item i

You should know how to define your own functions:

>>> def square(x):
...     return x*x

>>> square(5)
25

Above, x is a parameter and 5 is a positional argument that assigns a value to x. You should know about positional and keyword arguments. Note that the term parameter means something slightly different.

Each time a function is called, its parameters are new variables that exist only for that particular function invocation. A parameter variable has nothing to do with the argument variable you passed in, even if they have the same name like in the example below:

>>> def dummy(x):    # Argument variable 'x' will only exists when this function is actually called,
...     x = 1        # and is distinct from any other 'x' defined outside this function definition.
...     print(x)

>>> x = 99           # Right now the interpreter has only one 'x' variable, referring to integer 99.
>>> dummy(x)         # While 'dummy' is executing, the interpreter will have two 'x' variables.
1                    # Notice that the print(x) prints 1 and not 99
>>> print(x)         # When 'dummy' returns, there is only one 'x' variable again, and it hasn't changed.
99

For C programmers, think of function arguments as a pointer argument (int* not int):

You should know how to give your functions default arguments:

>>> def is_close(x, y, tolerance=0.001):
...    return abs(x-y) < tolerance

Never set an argument's default to a mutable object such as [] or {}. You will get very strange behaviour.

You should know how to return multiple values from a function:

>>> def swapped(x, y):     # When this function is called, unpack its arguments into variables 'x' and 'y'
...    return y, x         # Create a new tuple (y, x) and return it.

>>> a, b = swapped(5, 3)   # Create a new tuple (5, 3) to pass as the arguments, and unpack the result
>>> b
5

You should strive to know what the *args and **kwargs syntax means.

Lambda functions

You should know how to define a function using lambda notation:

>>> f = lambda x: x*x   # Create a function object and make a new variable 'f' that refers to it
>>> f
<function <lambda> at 0x0000023EEC452B88>

>>> f(5)                # Call the function object referred to by variable 'f'
25

You should know how to use lambdas as arguments to other functions:

>>> x = [1, -5, -3, 2]
>>> sorted(x)                         # Sort the list by directly comparing pairs x[i] < x[j]
[-5, -3, 1, 2]

>>> sorted(x, key=lambda y: y**2)     # Sort the list by comparing key(x[i]) < key(x[j]), where the 'key'
[1, 2, -3, -5]                        # argument refers to a function object taking one argument

Above, we could have used the absolute value (sorted(x, key=abs)) to achieve the same effect as squaring the key (y**2), even though abs is not a lambda function.

Types and Classes

Every object is an instance of some type. Machine learning libraries like PyTorch require you to define new types when expressing a custom model architecture.

You can check the type of a the object that a variable refers to:

>>> type(3)
<class 'int'>

>>> type(3.0)
<class 'float'>

>>> type('3')
<class 'str'>

You should know the basics of defining new types, of defining functions, and of storing variables:

>>> class Point2D:                  # Create a new type object called Point2D
...     def __init__(self, x, y):   # Define an initialization function that takes an (x, y) pair
...         self.x = x              # Create an attribute that refers to whatever object 'x' refers to
...         self.y = y              # Create an attribute that refers to whatever object 'y' refers to

>>> p = Point2D(3.5, 7.0)           # Create a new object of type Point2D, and run the __init__ function
>>> p
<Point2D object at 0x0000023EEC465FC8>

>>> p.x                             # Attribute 'x' on object referred to by 'p' now refers to the
3.5                                 # same float object that we passed as an argument to __init__

When a function is defined as part of a class, it is called a method.
When a variable is 'attached' to an object it is called an attribute.

You should also know what static methods are, and be able to explain why they has no self parameter:

>>> class Point2D:
...     def __init__(self, x=0.0, y=0.0):   # A method, since it needs reference to an instance (self)
...         self.assign(x, y)               # (delegate initialization to another method)
...
...     def assign(self, x, y):             # A method, since it needs reference to an instance (self)
...         self.x = x
...         self.y = y
...
...     @staticmethod
...     def dimensions():       # A static method; doesn't need a reference to any particular instance
...         return 2

>>> p = Point2D()               # Create a Point2D instance (0.0, 0.0) and make variable 'p' refer to it
>>> p.x                         # Attribute p.x refers to the float object used as the default for __init__
0.0

>>> p.assign(3.5, 7.0)          # Call the 'assign' method on whatever object 'p' refers to
>>> p.x                         # Attribute p.x now refers to the float object we passed as an argument
3.5

>>> Point2D.dimensions()        # Call the 'dimensions' method on the type object 'Point2D'
2                               # (Has nothing to do with the instance referred to by variable 'p')

A full tutorial on Python's type system is beyond the scope of this document. You should learn how to use classes in a general setting, such as the Python documentation on classes. If you only uses classes in the context of machine learning libraries, you may be able to struggle along by aping what you see, but you will probably not understand what's really going on.

The None object

You should know how the special None object is used in Python:

>>> x = None       # Make variable 'x' refer to the global None object
>>> x              # If we ask Python's REPL to evaluate None, it doesn't print anything
>>> print(x)       # But we can print the None object
None

It has similar usage as NULL in C programming, but Python variables are never actually NULL — they must always refer to some object (an integer, a string, something). So, there is one global object called "None" that any variable can refer to.

The None object is used to mean "no specific value." If a function has no return value, it returns None:

>>> def foo():
...    pass         # Pass just means "do nothing" in Python

>>> print(foo())    # The 'foo' function actually returns a reference to the global None object
None

Consider the dictionary get method, a useful alternative to dict[key] syntax:

>>> help(dict.get)
get(self, key, default=None)
    Return the value for key if key is in the dictionary, else default.

We can override default=None when calling the function.

>>> x = {'a': 0, 'b': 1}
>>> x['c']                  # If we access the value for a key that does not exist, an error is raised
KeyError: 'c'

>>> print(x.get('c'))       # If we 'get' the value for a key that does not exist, None is returned
None

>>> x.get('c', default=0)   # Specify a different object to return on missing keys, rather than None
0

Note that None evaluates to False when used in conditionals:

>>> bool(None)
False

This is relevant to if-statements that may evaluate a None object:

>>> if x.get('c'):          # get() returns a reference to the None object, which converts to False
...     print("Dictionary key 'c' was found")
... else:
...     print("Dictionary kKey 'c' was not found")
Dictionary key 'c' was not found

Since other objects may evaluate to False (such as the integer 0 or an empty string '') it is good practice to explicitly test whether a value is not None:

>>> if x.get('a') is not None:      # Evaluates to True even if get() returns 0 or False
...     ...

Docstrings

You should understand how functions and classes are documented by "docstrings." Basically, if you define a string at certain points in the code, Python recognizes it as documentation:

>>> def square(x):
...     """Returns x squared."""      # If the first line of a function is a string, it is used
...     return x*x                    # as the function's docstring

>>> help(square)                      # Anyone can see the docstring by using the 'help' function
square(x)
    Returns x squared.

>>> square.__doc__                    # The docstring is attached to the function object as an attribute
'Returns x squared.'

You should strive to follow the PEP257 docstring conventions.

Importing modules

The standard Python modules are very useful. There is too much to know, but some you will use frequently, such as the os module. In this course you will also use modules like numpy, sklearn, and torch.

You need to know different ways of importing modules and parts of modules:

>>> import os                        # Make a variable 'os' that refers to the module object called 'os'
>>> import os.path as path           # Make a variable 'path' that refers to the 'os.path' submodule
>>> from os import path              # Same as above
>>> from os import path as os_path   # Make a variable 'os_path' that refers to the 'os.path' submodule

The above import syntax is all doing pretty much the same thing, which is to make a variable that we can use to access a module's functionality, including get access to its submodules.

>>> type(os)               # The 'os' variable refers to a module object
<class 'module'>
>>> type(os.path)          # The 'path' attribute on that module object is also a module object
<class 'module'>
>>> os_path is os.path     # 'os_path' refers to exact same module object as the 'os.path' attribute
True

For example, the os.path submodule lets us check whether files and directories exist:

>>> help(os.path.exists)
exists(path)
    Test whether a path exists.  Returns False for broken symbolic links

>>> os.path.exists('.')
True
>>> os.path.exists('./foofoo')
False

Writing your own modules and packages is important for publishing code, but is not necessary for the course. (A 'package' is just a collection of one or more modules that are bundled together for distribution.)

Files

You should know how to open files for reading/writing and know when to use text versus binary mode:

  • text mode is for opening files that contain textual data, such as .txt or .csv;
  • binary mode is for opening all other files, such as .gzip or .pkl (pickle files).

Text mode is the default, so forgetting to use binary mode is a common source of confusing errors.

You should also know how to use the with-statement when working with files. (Students who know TensorFlow will have seen with-statements in other contexts as well.)

For example, imagine you have a file test.txt that contains the following:

One
Two
Three

(The third line is assumed to end with a newline, too.)

You can read a file all at once, as a big string:

>>> with open('test.txt') as f:     # Open a file and make a variable 'f' refer to the file object
...    s = f.read()                 # Make 's' refer to the string object returned by the 'read' method
...                                 # Exit the with statement, automatically closing the file
>>> s
'One\nTwo\nThree\n'

Notice that f still hangs around, but the file object it refers to was closed on exiting the with-statement:

>>> f.closed
True

You can read a text file into a list of strings (one per line) all at once:

>>> with open('test.txt') as f:
...    lines = f.readlines()
>>> lines
['One\n', 'Two\n', 'Three\n']

(Note that the newline characters are retained. Use the rstrip method if you don't want them.)

Or you can read a text file incrementally, line-by-line, using the file object as an iterator:

>>> with open('test.txt') as f:
...    for line in f:
...        print(line.rstrip())    # Using rstrip since 'print' already prints a newline
One
Two
Three

You can write a new text file by opening it for writing:

>>> with open('write.txt', 'w') as f:   # This time open for writing mode ('w')
...     for line in lines:              # Assume we still have lines = ['One\n', 'Two\n', 'Three\n']
...         f.write(line)               # Write the string object to the file

Serialization

Once you know files, you should know how to store Python objects using the pickle module.

>>> import pickle

It is useful for storing results so that they can be loaded later for analysis:

>>> result = {'loss': 0.0142, 'accuracy': 0.75}   # Create an object refering to whatever we want to store
>>> with open('result.pkl', 'wb') as f:           # Open a file for writing in binary mode
...     pickle.dump(result, f)                    # Serialize the object to the file using pickle format

>>> with open('result.pkl', 'rb') as f:           # Later on, open the file for reading in binary mode
...     result = pickle.load(f)                   # Deserialize the object from the pickle file
>>> result
{'loss': 0.0142, 'accuracy': 0.75}                # The object and all objects it refers to are re-created

Beware loading pickle files that were sent to you by someone else. Loading a pickle file can execute arbitrary code on your computer, so a nasty person could design one to delete all your files or to install a virus.

Exceptions

It is good to know how exception handling works. It works similarly as in C++ and Java. In this course, understanding Python exceptions is optional. Still, in a job interview you may be asked about them.

For example, if you open a file that does not exist, the open function will create an exception object of type FileNotFoundError and will raise it up the call stack. By default, your program will then terminate:

>>> with open('file_does_not_exist.txt') as f:       # Call open() with a file name that does not exist
...     print("Opened the file")
FileNotFoundError: No such file or directory: 'file_does_not_exist.txt'   # FileNotFoundError was raised

If you catch an exception object before it propagates all the way up the call stack, you have the option of responding to it differently:

>>> try:                                             # Begin a try-except block
...     with open('file_does_not_exist.txt') as f:
...         print("Opened the file")
... except:                                          # If an exception is raised, don't let it propagate
...     print("Exception caught!")                   # Instead, just print a message and continue
Exception caught!

Memory management

Python tries to manage memory automatically. Usually this works well. The only times it doesn't work well are when you're allocating huge objects that use up almost all available memory. This can happen in deep learning, but is still pretty rare. In this course, understanding Python memory management is optional. Still, in a job interview you may be asked to demonstrate a basic understanding.

Python uses two strategies:

  • Reference counting: if no variables or attributes refer to an object, free it immediately;
  • Garbage collection: the interpreter will periodically check for objects that can no longer be 'reached' by your program, and will eventually get around to freeing them.

When a Python decides that an object's memory can be freed, it will first call its special __del__ method:

>>> class Dummy:
...     def __init__(self): print("Create", self)
...     def __del__(self): print("Delete", self)

Printing within the __del__ method lets us see reference counting in action:

>>> d = Dummy()              # Make a new Dummy object and make variable 'd' refer to it (refcount is 1)
Create <Dummy object at 0x000001C830B02088>

>>> d = None                 # Make 'd' refer to something else, reducing the Dummy object's refcount to 0
Delete <Dummy object at 0x000001C830B02088>       # This causes its memory to be automatically freed

However, if two or more objects have attributes that refer to one another, this creates a cycle and Python's reference counting strategy fails to free the objects after we're done with them:

>>> x, y = Dummy(), Dummy()                        # Create dummy objects 1 and 2 (both refcount 1)
Create <Dummy object at 0x000001C830B020C8>
Create <Dummy object at 0x000001C830B02308>

>>> x.other = y     # Make new attribute 'x.other' refer to dummy object 2 (refcount now 2)
>>> y.other = x     # Make new attribute 'y.other' refer to dummy object 1 (refcount now 2)

>>> x = None        # Remove our only direct reference to dummy object 1 (refcount now 1)
>>> y = None        # Remove our only direct reference to dummy object 2 (refcount now 1)
>>>                 # Because there is still a "reference cycle", both dummy objects still use memory
                    # (no "Delete" messages), even though we can no longer access either object

Even in the above scenario, the garbage collector will eventually detect that the two dummy objects are "garbage", and will free their memory. Python does not have permanent memory leaks the way C does.

We can force the garbage collector to run to make a point about how it works, though this is not something you should normally be doing in your code:

>>> import gc        # Import the garbage collection module
>>> gc.collect();    # Force the garbage collector to do a sweep over memory
Delete dummy <Dummy object at 0x000001C830B020C8>      # Garbage collector frees memory for dummy object 1
Delete dummy <Dummy object at 0x000001C830B02308>      # Garbage collector frees memory for dummy object 2
